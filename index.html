酷Q + CoolQ HTTP + Nonebot

百度已经有很多教程了，我就总结一下自己的过程(有点忘了具体过程，可能会出现错误)

关于这三个东东的介绍

因为服务器系统是Ubuntu，所以在docker上跑wine-coolq，那么首先安装docker(root用户)

wget -qO- https://get.docker.com/ | SH

启动docker后台服务

sudo service docker start

测试hello world

docker run hello-world

没问题后，CoolQ HTTP文档有打包好了(有cqhttp插件)的docker镜像

然后访问 http://<你的IP>:9000/ 进入 noVNC（默认密码 MAX8char），登录 酷Q，即可开始使用（插件已自动启用，配置文件也根据启动命令的环境变量自动生成了）。一般情况下，你不太需要关注插件是如何存在于容器中的。

成功运行酷q后，用Nonebot框架扩展自己的需求

配置：
酷 Q 的 app\io.github.richardchien.coolqhttpapi\config\ 目录中应该已经有了一个名为 .json 的文件（ 为你登录的 QQ 账号）。修改这个文件，添加如下配置项，并且重启一下酷 Q

data\app\io.github.richardchien.coolqhttpapi\config.ini(没有就自己创一个)

用Python根据Nonebot编程：这里有简单的介绍

工具

  • git
  • github账号
  • node.js

Git是一个版本工具，此处用来将你的修改推送到你的github上(需要使用SSh赋予你的电脑推送东西到github的权限)

hexo博客生成工具，为什么用它，彬彬推荐的，我不懂hhh。它可以自动生成html+css+js，你不需要再去构造你blog的形状。
